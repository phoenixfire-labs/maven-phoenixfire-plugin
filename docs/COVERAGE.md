# Code coverage

JaCoCo is **off by default** (`jacoco.skip=true`). Enable the **95% line / 90% branch** aggregate gate on production modules with the **`coverage`** profile (or `-Djacoco.skip=false`).

## Scope

| Module | Included in gate |
|--------|------------------|
| `phoenixfire-api` | Yes |
| `phoenixfire-core` | Yes |
| `phoenixfire-fork-runner` | Yes |
| `phoenixfire-maven-plugin` | Yes |
| `phoenixfire-it` | No (Invoker harness only; `jacoco.skip=true`) |

End-to-end Invoker projects under `phoenixfire-it/src/it` exercise the plugin in real Maven builds but are not part of the JaCoCo bundle. Unit tests in the modules above are what drive the gate.

## Local commands

```bash
# Unit tests + coverage report + coverage check (no Invoker ITs)
mvn clean verify -Pcoverage

# Full CI parity (Invoker ITs + coverage check + install)
mvn clean verify install -Prun-its -Pcoverage
```

Open the aggregated HTML report:

`phoenixfire-coverage/target/site/jacoco-aggregate/index.html` (generated by the terminal `phoenixfire-coverage` module, which runs after all production modules in the reactor)

Per-module reports (after `test`):

`phoenixfire-core/target/site/jacoco/index.html` (and sibling modules).

## CI

Pull-request CI runs with **`-Pcoverage`** (JDK 17 and 21). See [DEVELOPMENT.md](DEVELOPMENT.md) for workflows and artifacts.

Local `mvn verify`, `mvn install`, and `mvn install -DskipTests` do **not** run the coverage gate unless you add `-Pcoverage`.

## Notes

- Coverage is **unit-test** coverage in the Maven JVM, not line hits inside forked test workers spawned during Invoker runs.
- The gate is **95%** line and **90%** branch coverage for the aggregate bundle. If the build fails at `jacoco-check`, open the aggregate report and add or extend tests in the module that shows gaps.
